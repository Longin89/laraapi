# Laravel Product API

Тестовое задание, реализующее CRUD по REST API с использованием кэширования (Redis).

## 🚀 Быстрый старт

### Запуск проекта
Проект развернут с использованием Docker.
Для запуска - склонируйте проект в удобную для Вас папку
```bash
git clone git@github.com:Longin89/laraapi.git
```
Переименуйте 2 файла 
```bash
.env.example
```
в
```bash
.env
```
(в корне проекта и в папке www) - они содержат все необходимые настройки.

Находясь в папке проекта, соберите его:

```bash
docker-compose build
```
и запустите все сервисы:
```bash
docker-compose up -d
```

### Установка зависимостей
Чтобы установить все зависимости проекта, перейдите в консоль контейнера php-fpm:
```bash
docker exec -it php-fpm bash
```
и выполните:
```bash
composer install
```

## 🛠 Работа с базой и с Postman

### 0. Наполнение базы с помощью сидеров
Не выходя из консоли php-fpm, выполните:
```bash
php artisan migrate:fresh --seed
```

после этого, можно проверить работоспособность проекта, перейдя в браузере по адресу:
```bash
http://localhost
```
в случае, если Вы видете надпись "File not found" - дайте полные права на папку с проектом.

### 1. Создание юзера и получение токена
- **Создание пользователя**: Отправьте `POST` запрос на:
`http://localhost/api/register`
В теле запроса (JSON) укажите `name`, `email`, `password` и `password_confirmation`, например:
```bash
{
    "name":"test",
    "email":"tst@mail.ru",
    "password":1122,
    "password_confirmation":1122
}
```
В ответ Вы получите информацию о новом пользователе, вида:
```bash
{
    "user": {
        "name": "test",
        "email": "tst@mail.ru",
        "updated_at": "2026-05-04T09:00:25.000000Z",
        "created_at": "2026-05-04T09:00:25.000000Z",
        "id": 2
    },
    "token": "1|tdu8JNDUTpybyUI3Wlc2OKKarfljorEfHPzItZAT675cd93b"
}
```
- **Использование токена**: Для защищенных методов (создание / обновление / удаление продуктов) добавьте заголовок `Authorization: Bearer {token}`.
Для просмотра списка продуктов или отдельно взятого продукта авторизация не нужна.

### 2. Управление продуктами
Для выполнения этих действий необходимо использовать токен:

- **Создание продукта**: 
`POST`
`http://localhost/api/products`
  Передайте необходимые поля продукта в JSON теле запроса, например:
```bash

{
    "name":"test_product",
    "price":"131.0",
    "category_id":"1"
}
```
- **Изменение продукта**:
`PUT` или `PATCH`
`http://localhost/api/products/{id}`
  Укажите ID продукта в URL и обновленные данные в теле запроса, например:
```bash

{
    "name":"test_product_updated",
    "price":"132.0",
    "category_id":"2"
}
```

- **Удаление продукта**:
`DELETE` 
`http://localhost/api/products/{id}`
  Отправьте запрос на удаление по ID продукта.

При неверном заполнении полей или отсутствии авторизации ответом будет JSON с соответствующей ошибкой.

Так-же следует учитывать, что каждый юзер может реализовывать операции только над теми продуктами, создателем которых он является, т.е. юзер не может удалять / изменять чужие товары - в ином случае Вы получите соответствующую ошибку в JSON.

## 🔓 Публичные методы (без аутентификации)

Следующие методы доступны всем пользователям без необходимости вводить токен:

- **Просмотр всех продуктов**: `GET /api/products`
  Метод поддерживает пагинацию, фильтрацию и сортировку.

  **Пагинация**: используйте параметр `page`.
  Пример: `http://localhost/api/products?page=1`

  **Фильтрация**:
  - `category_id`: фильтрация по ID категории.
  - `price_min`: минимальная цена.
  - `price_max`: максимальная цена.
  - `name`: поиск по названию (LIKE).

  **Сортировка**:
  - `sort`: поле для сортировки (`price`, `created_at`). По умолчанию: `created_at`.
  - `order`: направление сортировки (`asc` - по возрастанию, `desc` - по убыванию). По умолчанию: `asc`.

  **Примеры запросов**:
  - Фильтрация по категории и диапазону цен:
    `http://localhost/api/products?category_id=1&price_min=100&price_max=500`
  - Поиск продукта по названию с сортировкой по цене (от дорогих к дешевым):
    `http://localhost/api/products?name=iphone&sort=price&order=desc`
- **Просмотр одного продукта**:
`GET`
`http://localhost/api/products/{id}`

  Позволяет получить детальную информацию о конкретном продукте по его идентификатору.

## 🧪 Тестирование

### Как запустить тест
Для запуска теста используйте команду:
```bash
docker exec php-fpm php artisan test
```

 `ProductServiceCacheTest` проверяет корректность работы кэширования в сервисе продуктов.
 
 Важный нюанс: тест использует трейт ```RefreshDatabase```, который полностью очищает базу (для чистоты теста) - имейте это ввиду. Если после отработки теста Вам вновь понадобится заполненная база данных - выполните сидирование по новой.
